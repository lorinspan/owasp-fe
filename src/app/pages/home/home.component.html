OWASP Security Checker Application

Examen Cybersecurity Developer, Șpan Lorin, martie 2025.

Ce poți testa în această aplicație?

Această aplicație demonstrează vulnerabilități din OWASP Top 10 și permite testarea unor atacuri comune asupra aplicațiilor web.

Broken Access Control

Testăm problemele de control al accesului care permit utilizatorilor neautorizați să acceseze resurse restricționate.

  • Escaladarea privilegiilor – Un utilizator obișnuit poate obține acces administrativ.
  • Acces neautorizat la conturi – Posibilitatea de a accesa contul altui utilizator prin schimbarea unui ID.
  • Lipsa restricțiilor pe acțiuni importante – Permite ștergerea sau modificarea datelor altor utilizatori.

Cross-Site Scripting (XSS)

Explorăm atacurile XSS, unde input-ul nesigur este injectat în pagină pentru a executa scripturi malițioase.

  • Stored XSS – Codul malițios este salvat permanent în baza de date și afectează alți utilizatori.
  • Reflected XSS – Atacatorul introduce cod malițios printr-o solicitare HTTP, care este executată imediat.
  • DOM-based XSS – Manipularea nesigură a DOM-ului permite rularea de cod JavaScript injectat.
  • Stocarea parolelor in clar – Parolele sunt extrase in clar utilizand request catre backend.

Broken Authentication

Demonstrăm problemele legate de autentificare care pot permite atacatorilor să compromită conturi utilizator.

  • Parole slabe sau ușor de ghicit – Permite atacuri brute-force și credential stuffing.
  • Stocarea sesiunii în client-side nesecurizat – Cookie-uri și token-uri nesigure.
  • Lipsa rate limiting la login – Permite încercări nelimitate de autentificare.
  • Transmiterea parolelor în text simplu – Lipsa criptării face parolele ușor de interceptat.
  • Omiterea invalidării sesiunii la logout – Sesiunile rămân active după deconectare.

Cryptographic Failures + CSRF

Demonstrăm problemele legate de criptografie care pot duce la expunerea datelor sensibile.

  • Utilizarea algoritmului MD5 – Vulnerabil la atacuri rainbow table și brute-force.
  • Hardcoding de chei criptografice – Un atacator poate extrage cheia și decripta datele.
  • Stocarea parolelor în mod nesigur – Parolele sunt salvate într-un mod care le face ușor de spart.
  • Atac CSRF prin schimbarea adresei de e-mail – Atacatorii pot trimite o cerere CSRF pentru a modifica adresa de e-mail a utilizatorului fără consimțământul acestuia, ducând la pierderea controlului asupra contului.

Server-Side Request Forgery (SSRF)

Explorăm atacurile SSRF, unde utilizatorul poate introduce o adresă URL nesigură, iar serverul va efectua o solicitare către acea adresă fără nicio validare.

  • Manipularea cererilor către backend – Aplicația permite utilizatorului să introducă un URL extern pentru verificarea stocului, dar nu limitează domeniile accesibile.
  • Acces la resurse interne – Un atacator poate introduce o adresă URL internă (de exemplu, http://localhost:8080/api/bac/admin/config) pentru a accesa informații restricționate.
  • Ocolirea mecanismelor de autentificare – Deoarece serverul execută cererea direct, poate accesa date interne fără a necesita autentificare.
  • Expunerea informațiilor sensibile – Permite extragerea datelor administrative sau utilizatorilor prin exploatarea SSRF.

Path Traversal

Explorăm atacul Path Traversal, unde un atacator poate accesa fișiere interne ale serverului prin introducerea unor căi nesecurizate.

  • Citirea fișierelor sensibile – Utilizatorul poate introduce numele unui fișier pentru a vizualiza conținutul acestuia.
  • Exploatarea fără protecție – Endpoint-ul de backend permite utilizatorului să trimită o cale personalizată, fără restricții.
  • Accesarea fișierelor interne – Un atacator poate introduce ../../supersecretfolder/supersecretfile pentru a citi fișiere din afara folderului permis.
  • Expunerea datelor confidențiale – Atacul permite accesul la configurații, parole, chei API și alte date sensibile.

Remote Code Execution (RCE)

Explorăm atacurile de tip Remote Code Execution (RCE), unde un atacator poate executa comenzi malițioase la distanță pe server, exploatând o vulnerabilitate în aplicație.

  • Executarea comenzilor la distanță – Atacatorii pot trimite comenzi malițioase către server, care vor fi executate pe serverul de aplicație.
  • Vulnerabilități de tip input nesigur – Aplicațiile care permit utilizatorilor să introducă date nesigure (de exemplu, input de tip text) fără a le valida sau filtra pot expune serverul la RCE.
  • Accesibilitatea pentru utilizatori non-admini – În mod neașteptat, unii utilizatori non-admini pot exploata aceste vulnerabilități, ceea ce le permite să execute comenzi pe server, având acces la date sensibile sau la funcții administrative.
  • Injecție de comenzi – Aplicațiile care permit injecția de comenzi, de exemplu, prin execuția comenzilor sistemului, pot fi ușor exploatate de atacatori pentru a compromite serverul.
  • Escaladarea privilegiilor – Chiar dacă un atacator are privilegii de utilizator, exploatarea unei vulnerabilități de RCE poate duce la escaladarea privilegiilor, oferindu-i acces complet la server.
  • Executarea de malware – Atacatorii pot folosi RCE pentru a instala malware pe server, care poate compromite întreaga infrastructură și poate duce la furtul de date sau la controlul serverului de la distanță.
